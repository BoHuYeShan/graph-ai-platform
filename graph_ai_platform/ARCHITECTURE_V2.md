# PyGraph 架构重建设计稿（待 GPT 讨论）  ## 1. 当前问题和根因  ### 1.1 值不可编辑 `print(123)` 的 IR 结构： ``` name("print") ──callable──→ call ──arg_0──→ literal(123) ```  在 Blueprint 中，`literal(123)` 是一个节点，显示 "123" 但**不可点击编辑**。 在 Blockly 中，`123` 虽然显示在 `math_number` 块里，但改动后触发循环或丢失。  **本质问题**：图是代码的只读投影，不是双向编辑器。  ### 1.2 只有左→右，没有右→左 当前数据流： ``` CodeMirror → debounce → /parse → IR → GraphView/BlocklyEditor ```  缺少反向： ``` GraphView编辑(拖拽/连线/改值) → IR变化 → codegen → CodeMirror更新 ```  ### 1.3 端口无类型 ``` name("print") ──────────→ call literal(123) ───────────→ call ```  两条边没有类型区分。在蓝图中应该： - `callable` 端口（绿色）只接受函数名/可调用对象 - `arg_0`, `arg_1` 端口（红色/蓝色）只接受值参数 - 不同类型的端口不能互连  ### 1.4 参数多端口 `print(123, "hello", x)` 需要： ``` call   ├── callable ← name("print")   [绿色端口]   ├── arg_0    ← literal(123)    [红色端口]   ├── arg_1    ← literal("hello")[蓝色端口]   └── arg_2    ← name("x")      [蓝色端口] ```  当前只有 1 个输入端口，多个参数没有区分。  ## 2. 蓝图重设计  ### 2.1 节点类型系统  每个节点有**类型化端口**（socket），用颜色标识：  | 端口类型 | 颜色 | 用途 | 示例 | |---------|------|------|------| | `callable` | 🟢 绿 | 函数/方法引用 | `print`, `math.sqrt` | | `value` | 🔴 红 | 任意值 | 字面量/变量/表达式结果 | | `test` | 🟡 黄 | 布尔条件 | `n > 0`, `x == True` | | `iterable` | 🟣 紫 | 可迭代对象 | `[1,2,3]`, `range(10)` | | `target` | 🔵 蓝 | 赋值目标 | 循环变量 | | `body` | ⬜ 灰 | 语句体（容器） | if/while/for 内部 |  ### 2.2 节点定义  #### 函数调用 (`call`) ``` ┌──────────────────────────────────┐ │  📞 print()                      │ │  🟢 callable ───── name("print") │ │  🔴 arg_0     ───── 123          │ │  🔴 arg_1     ───── "hello"      │ │  🔴 arg_N     ───── ...          │ │  🟣 result    ──── (optional)    │ └──────────────────────────────────┘ ```  每个参数是一个独立的端口，可拖拽连线。参数数量动态（像 Unreal Blueprint 的 `Add Pin`）。  #### 赋值 (`assign`) ``` ┌────────────────────────────┐ │  x = ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓     │ │  🔴 value ──── 42         │ └────────────────────────────┘ ```  左边变量名可编辑（双击改名），右边值通过端口连接。  #### If 分支 ``` ┌──────────────────────────────────┐ │  🟡 if test ──── n > 0          │ │  ┌─ body ──────────────────────┐ │ │  │  [return "positive"]        │ │ │  └─────────────────────────────┘ │ │  ┌─ orelse ────────────────────┐ │ │  │  [return "non-positive"]    │ │ │  └─────────────────────────────┘ │ └──────────────────────────────────┘ ```  C 形容器，body/orelse 是子画布。  #### While 循环 ``` ┌──────────────────────────────────┐ │  🟡 while test ──── n > 0       │ │  ┌─ body ──────────────────────┐ │ │  │  [result *= n]              │ │ │  │  [n -= 1]                   │ │ │  └─────────────────────────────┘ │ └──────────────────────────────────┘ ```  ### 2.3 值编辑  字面量节点的值应该直接在节点上可编辑：  ``` ┌───────────────┐ │  123          │  ← 双击数字直接打字修改 │  🔴 out       │ └───────────────┘ ```  ``` ┌───────────────────────┐ │  "hello world"        │  ← 字符串被引号包裹，双击编辑 │  🔴 out               │ └───────────────────────┘ ```  ``` ┌───────────────┐ │  True         │  ← 布尔值，点击切换 True/False │  🔴 out       │ └───────────────┘ ```  ### 2.4 双向同步  ``` CodeMirror ←────────────────────────┐     ↓ (debounce 600ms)              │   /parse                            │     ↓                               │   IR (graph-ai.json) ──→ GraphView  │     ↓                               │   用户编辑图(拖拽/连线/改值)         │     ↓                               │   IR 更新 ──→ /codegen ──→ CodeMirror ```  **循环保护**： - CodeMirror → IR → Graph：用 `codeRef` 比较，相同不触发 - Graph → CodeMirror → IR：`codegen` 输出与当前 `code` 比较，相同不触发重新 parse - 用户操作图时，锁定 IR 更新方向（只 Graph→Code）  ## 3. 拼图模式 (Blockly) 重设计  ### 3.1 值可编辑 - `math_number` 块：点击数字直接键盘改 - `text` 块：字符串可编辑 - `logic_boolean`：下拉切换 True/False - 变量名：点击直接重命名  ### 3.2 表达式嵌套 ``` ┌──────────────────────┐ │  print               │ │  ┌────────────────┐  │ │  │  123           │  │  ← 值块嵌套在参数槽内 │  └────────────────┘  │ └──────────────────────┘ ```  `math_arithmetic` 块应支持多个操作数扩展： ``` ┌──────────────────────────────┐ │  ┌────┐ ┌────┐ ┌────┐      │ │  │ 1  │+│ 2  │+│ 3  │  ... │  ← 动态 + 号 │  └────┘ └────┘ └────┘      │ └──────────────────────────────┘ ```  ### 3.3 双向同步 与蓝图同样的双向机制，但通过 Blockly 的 `workspaceToCode` 生成 Python： ``` Blockly编辑 → workspaceToCode → code → CodeMirror                                             ↓ (debounce)                                           /parse → IR → 更新 Blockly（带 Events.disable 保护） ```  ## 4. 推荐实施路径  ### Step 1: Port 类型系统 - 在 `graph-ai.json` schema 中为每个端口添加 `type` 字段 - 定义端口类型枚举：`callable | value | test | iterable | target | body` - Blueprint 渲染时按类型着色  ### Step 2: 节点值内联编辑 - `literal` 节点：点击值直接修改 - `name` 节点：双击变量名重命名 - 实时同步到 IR → codegen → CodeMirror  ### Step 3: 多参数端口 - `call` 节点动态生成 `arg_N` 端口 - `binop` 节点支持扩展操作数（A + B + C） - 端口数量根据实际参数动态  ### Step 4: 双向同步完整链路 - Graph → /codegen → CodeMirror 的反向通道 - 循环保护（比较新旧代码/IR） - 用户拖拽连线、增删节点后自动同步  ### Step 5: 画布交互（拖拽/连线） - Blueprint：React Flow 自带拖拽连线 - 连线时按端口类型校验（绿色只能连绿色） - 删除节点时自动清理关联边 - 新增节点从工具箱拖入
